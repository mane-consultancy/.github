# Contributing

Thank you for contributing.

## Before You Start

- Make sure the scope of your change is clear.
- Prefer small, focused pull requests.
- Open or reference an issue when the change needs discussion or tracking.

## Development Expectations

- Follow the conventions already used in the repository.
- Keep changes readable and maintainable.
- Add or update tests when relevant.
- Update documentation when behavior or usage changes.

## Pull Requests

- Describe what changed and why.
- Include validation details so reviewers can understand how the change was checked.
- Call out any risks, follow-up work, or breaking changes.

## Reviews

- Address feedback clearly and keep discussion attached to the pull request.
- If a suggestion is intentionally not applied, explain the reason.
